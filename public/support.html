AXIS Companion — Support

iOSPWA Pairing, calibration, OTA, screensaver, music. If anything is stuck, the answer is usually below.

Getting started

  • Power the AXIS knob (USB-C or installed power). The boot splash appears for ~1 s, then the wizard or last-used screen.
  • Open AXIS Companion on your phone. The first screen is Devices (or DEMO MODE if you don't have a knob handy).
  • Tap Scan. The AXIS knob appears in the list within ~5 s; tap to pair. iOS will ask once for Bluetooth permission — accept.

Common questions

The app is stuck on the splash / black screen.

Force-quit and reopen. If it persists, delete and reinstall — the app's only persistent state is the BLE device ID, which re-pairs automatically on next launch.

"Scan" finds nothing.

  • Bluetooth permission denied. Settings → AXIS Companion → allow Bluetooth.
  • AXIS is asleep — tap the knob once to wake it.
  • Another phone is currently paired. AXIS only accepts one host at a time; force-quit the app on the other phone, or power-cycle the knob.

"Connect" hangs / times out.

  • Toggle Bluetooth off then on in iOS Settings.
  • On the knob, 5-tap from the gear digit to reach the menu, scroll to BOOT, and confirm — a clean reboot clears stale BLE state on the device side.
  • As a last resort, delete the pairing record from iOS Settings → Bluetooth → tap "i" beside the AXIS entry → Forget This Device, then re-pair from the app.

Firmware update (OTA) gets stuck or fails.

  • WiFi creds wrong. Go to Sys → check the "WiFi" tile. Re-enter SSID + password if needed.
  • Knob isn't in range of WiFi. Move it closer to the router before tapping INSTALL.
  • HTTPS request to snmotorsport01.github.io is failing. Try again later; this is GitHub's CDN.
  • Bricked the OTA partition. Connect by USB-C and reflash using esptool.py write_flash 0x0 smart_knob_shifter-vX.Y.Z-merged.bin. The merged bin is downloadable from the OTA page in the app.

Calibration won't capture / says "HOLD STILL".

The IMU is integrating motion above the noise floor. Set the knob on a flat, vibration-free surface and tap again. If it persists, tap SET ZERO from the main menu to re-acquire the gyro bias.

Screensaver upload finishes but doesn't appear.

The decoder PSRAM allocation failed. Verify firmware is v2.5.40+ which fixes the PSRAM mode for the R2 module. If you're on v2.5.40+ and still hit this, the source image is likely too large — resize to under 240×240 before uploading.

Music mode shows WAITING instead of track info.

  • iPhone isn't paired as a media host. iOS will show a "Receive Bluetooth..." prompt the first time — accept it.
  • Music app is closed. AMS (Apple Media Service) only publishes metadata while a music app is in the foreground or playing in the background.
  • Album art missing: this is normal. AXIS looks the art up via MusicBrainz + Cover Art Archive over WiFi — niche or podcast tracks won't have a match, and the vinyl-spin fallback shows instead.

How do I switch screens on the knob?

  • Tap — context-specific (play/pause on Music, etc.).
  • 5-tap from the gear digit — opens the main menu.
  • Long-press — back to the previous screen or sleep.
  • Triple-tap — lock the touch. Long-press again to unlock.
  • Swipe — varies per screen (e.g., left/right on Main = gear pattern, on Music = volume).

Demo mode (no hardware required)

Tap DEMO MODE on the Devices screen to explore the app without an AXIS knob. The Live page renders a fake telemetry stream; Brand / Screensaver / Music pages preview UI without sending anything. All other pages show static demo data so the layout is visible. Useful for the App Store reviewer and for trying out customisations before buying the hardware.
